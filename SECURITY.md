# Security Policy

## Reporting

If you find a security issue in this repository, do not open a public issue first.

Report it with:

- A concise summary of the issue
- Affected files, scripts, or skills
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation if available

Use one of these channels:

- GitHub Security Advisories for this repository, if enabled
- A private maintainer contact listed by the repository owner

## Scope

This repository includes:

- Install, update, and uninstall scripts
- Skill and workflow definitions
- Marketplace discovery and merge logic

Security reports for remote third-party skill repositories should identify:

- The external repository URL
- The pinned ref or branch used
- Whether the issue exists in this repository or only in the upstream source

## Response Expectations

- Acknowledge receipt as soon as practical
- Validate impact before public disclosure
- Prefer coordinated disclosure after a fix or mitigation is ready
